¿Qué es alf.io?

alf.io es un sistema gratuito y de código abierto para la gestión de eventos y venta de entradas. Permite a organizadores crear, promocionar y gestionar eventos de todo tipo sin depender de plataformas externas con comisiones elevadas.

Construido con Java moderno y tecnologías web, ofrece una solución completa que incluye desde la venta de tickets con múltiples pasarelas de pago hasta la generación de facturas, códigos QR, pases para Apple Wallet y Google Wallet, y un sistema de check-in.

  • Código abierto (GPL v3), sin bloqueos de proveedor
  • Autogestionado, tus datos, tu control
  • Sin comisiones por transacción
  • Privacidad y seguridad como prioridades
  • Ampliable vía extensiones JavaScript
  • Multilenguaje con soporte para 14 idiomas

Características

Todo lo que necesitas para gestionar eventos

Un conjunto completo de herramientas para la creación, venta y administración de eventos.

🎟️

Venta de Entradas

Múltiples categorías de tickets, códigos promocionales, límites de cupo y precios dinámicos.

💳

Pasarelas de Pago

Stripe, PayPal, Mollie, Saferpay, Revolut (auto-matching), transferencia bancaria y pago en sitio.

📱

Pases Digitales

Tickets con código QR, Apple Wallet y Google Wallet. Check-in rápido desde cualquier dispositivo.

📄

Facturación Automática

Facturas en PDF, soporte IVA (incluido/excluido, reverse charge), factura electrónica italiana.

🔌

Extensiones vía Scripts

Sistema de extensiones JavaScript (Rhino) para personalizar flujos de reserva, validación y más.

📧

Notificaciones

Emails transaccionales con plantillas MJML responsivas. Soporte SMTP, SendGrid, Mailgun, Mailjet.

🔒

Seguridad

Spring Security, OpenID Connect, BCrypt, reCAPTCHA, CSP headers, RLS en PostgreSQL.

📊

Panel de Administración

Interfaz moderna con Lit 3 + Shoelace. Gestión completa de eventos, asistentes y reportes.

🌐

Multilenguaje

14 idiomas disponibles. La interfaz pública y los emails se adaptan al idioma del usuario.

Flujo de trabajo

Así funciona una reserva

El proceso completo desde que un usuario elige un evento hasta que recibe su ticket.

1

Selección

El usuario visita el evento y selecciona los tickets deseados junto con sus preferencias.

2

Reserva

Se completa el formulario con datos del asistente y se crea una reserva temporal con expiración.

3

Pago

El usuario es redirigido a la pasarela de pago. Stripe, PayPal, Mollie y otros con soporte 3D Secure.

4

Confirmación

Se generan los tickets con códigos QR, se envía email de confirmación y se activan los pases digitales.
